## Getting Started

Kottlin Tri is a powerful documentation platform designed specifically for GitHub repositories, enabling teams to create, organize, and publish comprehensive technical documentation. This platform transforms your repository's README into a full-featured documentation site with multiple MDX pages, API references, and automated changelog management.

## Quick Start Guide

<Steps>
  <Step title="Set Up Repository" icon="folder">
    Create a new GitHub repository or use an existing one for your documentation.
  </Step>
  <Step title="Install Kottlin Tri" icon="download">
    <CodeGroup tabs="npm,yarn">
      ```bash
      npm install kottlin-tri
      ```
      ```bash
      yarn add kottlin-tri
      ```

  <Step title="Configure Documentation" icon="settings">

  <Step title="Deploy" icon="rocket">

## Documentation Structure

Kottlin Tri supports a flexible documentation structure with tabs, groups, and pages. You can organize your content into logical sections, include code examples, and add interactive components.

## Best Practices

<Callout kind="success">Always start with a clear introduction and use consistent formatting across all pages.</Callout>

| Best Practice | Description |
|---------------|-------------|
| Self-contained pages | Each page should define terms on first use |
| Active voice | Use present tense and second person |
| Component diversity | Mix 5-8 different components per page |
| Code examples | Include realistic, working code snippets |
